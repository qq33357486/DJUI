# 安全说明

DJUI 是本地开发工具，不是公网服务。后端需要读取本机目录、复制 Runtime、浏览素材和发布文件，因此安全边界默认是“只给本机开发者使用”。

## 默认安全策略

后端默认：

- 监听 `127.0.0.1:37241`
- 只允许本机来源的 CORS 请求
- 不会自动杀掉占用端口的进程
- 图片接口只返回受支持的图片扩展名
- 页面 ID 和待审核素材路径会做路径穿越校验

前端默认：

- 监听 `127.0.0.1:7321`
- 通过 Vite proxy 访问 `http://localhost:37241/api`

Docker 镜像会设置 `DJUI_HOST=0.0.0.0`，这是为了让宿主机可以通过端口映射访问容器。运行容器时仍建议只映射到可信主机或本机。

## 环境变量

| 变量 | 默认值 | 说明 |
|---|---|---|
| `DJUI_PORT` | `37241` | 后端端口 |
| `DJUI_HOST` | `127.0.0.1` | 后端监听地址 |
| `DJUI_CORS_ORIGIN` | 空 | 额外允许的 CORS 来源，多个来源用逗号分隔 |
| `NODE_ENV` | 空 | `production` 时后端会 serve 前端 dist |

需要局域网访问时：

```powershell
$env:DJUI_HOST="0.0.0.0"
$env:DJUI_CORS_ORIGIN="http://你的前端地址:7321"
```

只在可信网络中这样做。不要把 DJUI 后端暴露到公网。

## 本机配置文件

`editor/backend/djui_config.json` 保存当前绑定的 StarEngine 工程路径和 UI 工作区路径。它是本机状态，已被 `.gitignore` 排除。

公开提交前请确认没有把以下内容提交到仓库：

- `editor/backend/djui_config.json`
- `.env`
- `node_modules/`
- `dist/`
- UI 工作区中的临时素材、原始大图和私有资源

## 路径与文件接口

编辑器后端提供目录浏览和图片预览接口，这是为了本地工具体验。相关接口不应该被未授权用户访问。

当前保护措施：

- `/api/assets/file` 只返回图片扩展名。
- `base + rel` 组合路径必须解析在 `base` 目录内。
- `/api/pages/:pageId` 拒绝包含路径分隔符、`..` 和 Windows 保留字符的页面 ID。
- 待审核素材批准/拒绝只接受 `临时文件/待审核/<分类>/<文件名>`。

## 发现安全问题

如果发现可以读取非图片文件、越权写入项目外路径、绕过 CORS 或破坏本机文件的漏洞，请优先通过 GitHub Security Advisory 或私下渠道告知维护者。公开 issue 中不要贴真实敏感路径或复现所需的私有文件内容。

## 二、原始素材规范

- **按日期管理**：每天创建一个日期目录，如 `原始素材/2026-06-28/`
- **可再细分**：日期目录下可按主题/批次建子目录，如 `原始素材/2026-06-28/home_bg/`
- 放入内容：原图、AI 生图、截图、参考图、PSD 等
- 这些素材**不参与发布**，仅作为加工来源
- 推荐保留原始高分辨率，压缩版归到 `临时文件/` 或丢弃

## 三、移动端素材尺寸规范（重要）

本项目目标平台是**手机游戏**。所有成品素材的最终分辨率必须遵守下面规则。
依据 Apple Human Interface Guidelines 和 Google Material Design 的移动应用贴图建议：

### 3.1 硬性上限

| 项 | 上限 | 说明 |
|---|---|---|
| 单张贴图最大边长 | **1024 px** | 任何素材都不应超过；超过即视为未完成 |
| 单文件大小 | **500 KB** | 超大背景可放宽至 1 MB |
| 2 的幂对齐 | 推荐 | 1024/512/256/128/64/32；非 2 的幂 PNG 也能用但纹理上传略慢 |

### 3.2 各类素材推荐尺寸

| 素材类型 | 推荐尺寸 | 说明 |
|---|---|---|
| 图标（icons） | **64 / 128 / 256** | 头像、道具、货币图标用 256；功能小图标用 64 或 128 |
| 按钮（buttons） | **256×128 或 512×128** | 横长按钮；方按钮用 256×256 |
| 列表项/卡片（lists） | **512×256** | 可拉伸的留 9 宫格安全区 |
| 边框（frames） | **256×256** | 九宫格切片框 |
| 装饰物（decorations） | **256×256** | 光效、花纹、角标 |
| 艺术字（text） | **512×256** | 标题图、Logo |
| 全屏背景（backgrounds） | **1024×1024**（或 1024×512 分区） | 全屏用九宫格拉伸，不要单文件塞 1920×1080 |
| 未分类（misc） | **512×512** | 默认规范尺寸 |

### 3.3 同一资源多尺寸

需要在不同设备密度下显示的图标/头像，输出多个尺寸版本，用尺寸后缀区分：

```
icons/icon_avatar_64.png
icons/icon_avatar_128.png
icons/icon_avatar_256.png
```

引擎侧按设备 DPR 选择加载哪个版本。

## 四、成品素材分类规范（重点）

成品素材是「发布到引擎」的**唯一来源**。请严格遵守：

### 4.1 总原则

1. **必须放入八大分类目录之一**，禁止直接放在 `成品素材/` 根目录
2. 引擎引用路径 = `image/djui/<分类>/<可选子目录>/<文件名>`
3. 命名：**全小写 + 下划线**，禁止中文、空格、大写字母
4. 命名前缀**必须等于分类目录名**（如 `icons/icon_coin.png`，不是 `icons/coin.png`）
5. 按钮多状态用后缀：`_normal` `_pressed` `_disabled` `_hover`
6. 尺寸必须符合 §三 规范（单边 ≤ 1024，按类型选推荐尺寸）
7. 单文件 < 500 KB（背景可放宽至 1 MB）
8. 格式优先级：`.png`（透明/需要锐利边缘）> `.webp`（照片/渐变）> `.jpg`（无透明的大背景）

### 4.2 分类决策树

拿到一张图，按下面顺序判断该放哪个分类：

```
是图标吗？（小尺寸、单色或少量颜色、表达"功能/物品/状态"语义）
  → icons/

是按钮吗？（用于点击触发操作，可包含 normal/pressed/disabled 多状态）
  → buttons/

是文字图片吗？（艺术字、Logo 标题、装饰性数字）
  → text/

是边框/容器底框吗？（对话框、卡片外框、九宫格切片框）
  → frames/

是列表项/卡片吗？（可复用的列表行背景、商品卡、技能卡底图）
  → lists/

是装饰物吗？（光效、花纹、角标、非交互的氛围元素）
  → decorations/

是整屏背景吗？（铺满整个屏幕的底图、分区大背景）
  → backgrounds/

都不是？→ misc/（并定期重新归类）
```

### 4.3 八大分类详细说明

| 分类目录 | 用途 | 命名模板 | 推荐尺寸 | 反例 |
|---|---|---|---|---|
| `backgrounds/` | 全屏背景、分区大背景、面板底图（铺满式） | `bg_<场景>.png` | 1024×1024 / 1024×512 | 把按钮塞进背景图 |
| `buttons/` | 可点击按钮（含多状态） | `btn_<功能>_<状态>.png` | 256×128 / 512×128 | `close.png`（缺前缀和状态） |
| `frames/` | 对话框外框、Tooltip 框、九宫格切片框 | `frame_<用途>.png` | 256×256 | 与 `lists/` 混淆 |
| `icons/` | 功能图标、道具图标、货币图标、头像 | `icon_<对象>[_<尺寸>].png` | 64/128/256 | 不加尺寸后缀的多版本 |
| `lists/` | 列表行背景、商品卡、技能卡底图 | `card_<用途>.png` 或 `list_<用途>.png` | 512×256 | 与 `frames/` 混淆 |
| `decorations/` | 光效、粒子贴图、花纹、角标（**非交互**） | `deco_<描述>.png` | 256×256 | 把按钮放这里 |
| `text/` | 艺术字标题、Logo、装饰性大数字 | `title_<场景>.png` 或 `logo_<品牌>.png` | 512×256 | 普通文字做成图（应用 Label 渲染） |
| `misc/` | 暂时无法归类的素材（应急缓冲区） | 自由命名，仍需小写+下划线 | 512×512 | 长期不清理 |

> `frames/` 是「外壳」，`lists/` 是「行/卡片内容底图」—— 容易混淆，按是否包内容判断。

### 4.4 命名标准（强制）

```
<分类前缀>_<语义名>[_<尺寸或状态>].png
```

| 规则 | 说明 | ✅ 正确 | ❌ 错误 |
|---|---|---|---|
| 全小写 | 禁止大写字母 | `icon_coin.png` | `Icon_Coin.png` |
| 下划线分词 | 禁止空格、连字符、中文 | `btn_confirm_normal.png` | `btn-confirm normal.png`、`按钮.png` |
| 前缀=分类 | 必须以分类名开头 | `icons/icon_coin.png` | `icons/coin.png` |
| 尺寸后缀 | 多尺寸版本必加 | `icon_coin_256.png` | `icon_coin_big.png` |
| 按钮状态 | 按钮必加状态后缀 | `btn_confirm_pressed.png` | `btn_confirm_2.png` |

## 五、标准加工流程（去绿幕 → 裁边 → 压缩）

`脚本区/` 提供了配套工具脚本，对应下面三个标准步骤。每批 AI 生图或截图都应走完整流程再进成品目录。

### 5.1 四步流程

```
原始素材/  ──[1.去绿幕]──▶  临时文件/去绿幕后/  ──[2.裁边+压缩+规范化]──▶  临时文件/待审核/<分类>/  ──[3.人工审批]──▶  成品素材/<分类>/
```

> **重要**：第 2 步脚本的 `--output-dir` 必须指向 `临时文件/待审核/<分类>/`，**禁止直接写入 `成品素材/`**。
> 所有素材必须经过第 3 步人工审批后才能进入成品目录。审批在 DJUI 编辑器的「待审核」面板中操作。

#### 第 1 步：去绿幕（`green_key_to_png.py`）

AI 生图或拍摄图常带绿幕背景，必须先抠出前景。

```bash
cd 脚本区
python green_key_to_png.py \
    --input-dir  ../原始素材/2026-06-28/batch01-绿幕 \
    --output-dir ../临时文件/去绿幕后
```

- HSV 绿色判定 + 边缘扩散 + BFS 连通边缘溢出清除
- 自动去掉文件名中的「绿幕」「green」等标记
- 不改变尺寸，只去背景

#### 第 2 步：裁边 + 压缩 + 尺寸规范化（`trim_compress.py`）

把去绿幕后的图规范成符合 §三、§四 的待审核素材。

```bash
# 输出到待审核目录（推荐，必须人工审批后才能进成品素材）
python trim_compress.py \
    --input-dir  ../临时文件/去绿幕后 \
    --output-dir ../临时文件/待审核/icons

# 或显式指定分类
python trim_compress.py \
    --input-dir  ../临时文件/去绿幕后 \
    --output-dir ../临时文件/待审核/buttons \
    --category buttons

# 自定义尺寸（覆盖分类默认）
python trim_compress.py \
    --input-dir  ../临时文件/去绿幕后 \
    --output-dir ../临时文件/待审核/backgrounds \
    --width 1024 --height 512 --padding 0
```

这一步会自动：
- alpha 裁边（去掉四周透明像素）
- 居中适配到分类目标尺寸画布（保留比例）
- PNG 量化压缩（默认 192 色）
- 输出文件名规范化（加分类前缀 + 尺寸后缀）

> **禁止将 `--output-dir` 直接指向 `成品素材/`**。所有脚本加工的素材必须先进 `临时文件/待审核/`。

#### 第 3 步：人工审批（在 DJUI 编辑器中操作）

打开 DJUI 编辑器的「待审核」面板，查看 `临时文件/待审核/<分类>/` 中的素材：
- 逐张查看缩略图，确认透明边缘干净（无绿边残留）
- 确认尺寸符合 §三 表格
- 确认文件名符合 §4.4 命名标准
- **批准**：素材移入 `成品素材/<分类>/`，可用于编辑和发布
- **拒绝**：素材被删除（不合格的请回到第 1 步重新加工）

### 5.2 脚本区维护

`脚本区/` 的脚本由 DJUI 编辑器统一维护。当 DJUI 仓库的 `scripts/` 升级后，编辑器会通过「检查工作区更新」提示同步到工作区的 `脚本区/`。
**不要**在工作区里手动改脚本，定制需求请改 DJUI 仓库 `scripts/` 的源码。

脚本版本号在工作区 `脚本区/version.txt`，与 DJUI 仓库 `scripts/version.txt` 比较。

## 六、完整工作流程

1. 将原始素材或 AI 生图放入 `原始素材/YYYY-MM-DD/`
2. 按 §五 四步流程加工：去绿幕 → 裁边+压缩+规范化 → 人工审批 → 进入成品素材
3. 第 2 步输出到 `临时文件/待审核/<分类>/`，第 3 步在编辑器中审批后移入 `成品素材/<分类>/`
4. 在 DJUI 编辑器中编辑 UI 布局
5. 点击「发布」，`成品素材/` 全部内容（含子目录）原样同步到星火工程的 `ui/image/djui/`
6. 引擎中通过 `image/djui/<分类>/<文件名>` 引用

## 七、发布行为

- 发布会**完整覆盖** `ui/image/djui/` 下同名文件
- 子目录结构原样保留：`成品素材/icons/abc.png` → `ui/image/djui/icons/abc.png`
- `原始素材/` `临时文件/` `脚本区/` **不会**被发布
- 发布前请关闭引擎对相关资源的占用（否则 Windows 可能锁文件）
